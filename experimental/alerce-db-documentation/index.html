ALeRCE Database ZTF / PostgreSQL

Technical reference for the ALeRCE (Automatic Learning for the Rapid Classification of Events) PostgreSQL database. This document covers the schema exposed under the alerce schema: 34 tables containing ZTF alert-stream objects, photometry, machine-learning classifications, extracted features, and multi-catalog cross-matches.

Scope This documentation was generated from a live introspection of the alerce schema (information_schema queries on 2026-03-27) and is intended as a quick-start reference for astronomers who need to write SQL queries against the ALeRCE database. For the REST API and the Python client wrapper, see the Python Client section.

Connection

The database is a PostgreSQL instance. Read-only credentials are published in a JSON file maintained by the ALeRCE team. The recommended connection pattern (Python + SQLAlchemy) is: 

import requests, sqlalchemy as sa

url = "https://raw.githubusercontent.com/alercebroker/usecases/master/alercereaduser_v4.json"
params = requests.get(url).json()['params']

engine = sa.create_engine(
    "postgresql+psycopg2://"
    + params["user"] + ":" + params["password"]
    + "@" + params["host"] + "/" + params["dbname"]
)
conn = engine.connect()
Performance note These credentials grant read-only access. Queries that scan large fractions of the detection or forced_photometry tables (hundreds of millions of rows) can be slow. Always filter by oid or use sub-queries to restrict the result set.

Classification Tables

probability

The unified classification probability table. Contains class probabilities from any classifier registered in the system. Each row stores the probability of one class for one object under one classifier version. The ranking column orders classes from most probable (rank 1) to least probable.

ColumnTypeDescription
oidFKvarcharZTF object identifier
classifier_namevarcharName of the classifier (e.g. lc_classifier, stamp_classifier)
classifier_versionvarcharVersion string (e.g. hierarchical_rf_1.1.0)
class_namevarcharPredicted class label (e.g. SNIa, LPV, AGN)
rankingsmallintRank of this class (1 = most probable)
probabilityrealProbability assigned to this class

lc_classifier and sub-tables

The light-curve classifier uses a Balanced Random Forest with a hierarchical two-level scheme. The top level separates objects into periodic, stochastic, and transient categories. The bottom level resolves 15+ specific classes. The database exposes these through several tables:

TableDescription
lc_classifierFull class probabilities for the production LC classifier
lc_classifier_topTop-level probabilities (periodic / stochastic / transient)
lc_classifier_periodicPeriodic sub-class probabilities (CEP, DSCT, EB, LPV, RRL, etc.)
lc_classifier_stochasticStochastic sub-class probabilities (AGN, Blazar, CV/Nova, YSO, etc.)
lc_classifier_transientTransient sub-class probabilities (SNIa, SNIbc, SNII, SLSN, etc.)
lc_classifier_betaBeta/experimental version of LC classifier
lc_classifier_bhrf_forced_photBalanced Hierarchical RF on forced photometry
lc_classifier_bhrf_forced_phot_topTop-level probabilities (forced phot)
lc_classifier_bhrf_forced_phot_periodicPeriodic sub-classes (forced phot)
lc_classifier_bhrf_forced_phot_stochasticStochastic sub-classes (forced phot)
lc_classifier_bhrf_forced_phot_transientTransient sub-classes (forced phot)
lc_classifier_bhrf_forced_phot_childsChild-level class probabilities (forced phot)

All these tables share the same schema: (oid, classifier_name, classifier_version, class_name, ranking, probability).

stamp_classifier

The stamp-based classifier operates on the science, reference, and difference image cutouts (stamps) using a convolutional neural network. It provides rapid classification shortly after the first alert. Same schema as probability. The table stamp_classifier_2025_beta holds results from the 2025 beta model.

SQL Examples

List all tables in the schema

SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'alerce'
ORDER BY table_name;

Select supernova candidates with probability > 0.5

This query retrieves all objects whose top-ranked class (from the lc_classifier) is one of the four supernova types, with probability above 0.5, then joins back to probability to obtain the full probability vector for each object. 

SELECT
    object.oid, object.meanra, object.meandec,
    object.ndet, object.firstMJD, object.deltajd,
    object.g_r_max,
    probability.classifier_name,
    probability.class_name,
    probability.classifier_version,
    probability.ranking,
    probability.probability
FROM
    object INNER JOIN probability
    ON object.oid = probability.oid
WHERE
    probability.classifier_name = 'lc_classifier'
    AND object.oid IN (
        SELECT oid
        FROM probability
        WHERE classifier_name = 'lc_classifier'
            AND class_name IN ('SNIa', 'SNIbc', 'SNII', 'SLSN')
            AND ranking = 1
            AND probability > 0.5
    );

Retrieve detections for a set of objects

Given a list of object IDs (e.g. from the previous query), retrieve the full light curve from the detection table. 

SELECT
    oid, candid, mjd, fid, magpsf, sigmapsf
FROM
    detection
WHERE
    oid IN ('ZTF17aaahhwn', 'ZTF22abkzwso')
ORDER BY
    oid, mjd;

Get forced photometry for a single object

SELECT
    mjd, fid, mag, e_mag, mag_corr, e_mag_corr,
    isdiffpos, dubious, diffmaglim
FROM
    forced_photometry
WHERE
    oid = 'ZTF18abbuksn'
    AND procstatus = '0'
ORDER BY mjd;

Cross-match an object with external catalogs

SELECT
    x.catid, x.oid_catalog, x.class_catalog,
    x.dist, x.period,
    a.w1mpro, a.w2mpro
FROM
    xmatch x
    LEFT JOIN allwise a
        ON x.oid_catalog = a.oid_catalog
WHERE
    x.oid = 'ZTF18abbuksn';

Python Client

The alerce Python package provides a high-level wrapper around the ALeRCE API, avoiding the need to write SQL for common operations. Install with pip install alerce. 

from alerce.core import Alerce
client = Alerce()

# Query all objects classified as LPV
lpv = client.query_objects(
    classifier="lc_classifier",
    class_name="LPV",
    format="pandas"
)

# Get the full light curve (detections + non-detections)
lc = client.query_lightcurve("ZTF18abbuksn", format="pandas")

# Retrieve forced photometry
fp = client.query_forced_photometry("ZTF18abbuksn", format="pandas")

# Get classification probabilities
probs = client.query_probabilities("ZTF18abbuksn")

# Get variability features
feats = client.query_features("ZTF18abbuksn", format="pandas")

# Get magnitude statistics
stats = client.query_magstats("ZTF18abbuksn")
